vm2 is a sandbox library for Node.js that lets you execute untrusted JavaScript code inside the same process as your application. It uses a deep network of ES6 Proxies to intercept every interaction between sandboxed code and the host environment, providing fine-grained control over what the sandbox can see and do.

## Choose your sandbox

vm2 ships two sandbox classes suited to different use cases.

<CardGroup cols={2}>
  <Card title="VM" icon="square-terminal" href="/concepts/vm">
    Synchronous execution without `require`. Ideal for expression evaluation, templating, and plugin logic where module loading is not needed.
  </Card>
  <Card title="NodeVM" icon="node-js" href="/concepts/nodevm">
    Full Node.js module environment with configurable `require` controls. Suitable for running complete scripts that need to load modules.
  </Card>
</CardGroup>

## How vm2 works

<Steps>
  <Step title="Install">
    Add vm2 to your project with your preferred package manager.

    ```bash
    npm install vm2
    ```
  </Step>
  <Step title="Create a sandbox">
    Instantiate `VM` or `NodeVM` with your security options.

    ```javascript
    import { VM } from 'vm2';

    const vm = new VM({
      timeout: 1000,
      allowAsync: false,
    });
    ```
  </Step>
  <Step title="Run untrusted code">
    Pass code strings or precompiled `VMScript` objects to `vm.run()`.

    ```javascript
    const result = vm.run('1 + 1'); // 2
    vm.run('process.exit()');       // Throws ReferenceError: process is not defined
    ```
  </Step>
</Steps>

<Warning>
  vm2 provides in-process sandboxing, not OS-level isolation. Read the [Security Disclaimer](/security-disclaimer) before using vm2 with untrusted code from unknown sources.
</Warning>
