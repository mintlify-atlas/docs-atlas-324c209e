vm2 is available on npm and works with Node.js 6 and later. This guide walks you through installation and the three main ways to use vm2: the `VM` class for simple expression sandboxing, `NodeVM` for sandboxes that need to load modules, and `VMScript` for precompiling code you run repeatedly.

<Warning>
  Read the [security disclaimer](/security-disclaimer) before using vm2 in production. vm2 provides in-process sandboxing, not OS-level isolation, and has known limitations.
</Warning>

## Installation

<CodeGroup>

```bash npm
npm install vm2
```

```bash yarn
yarn add vm2
```

```bash pnpm
pnpm add vm2
```

</CodeGroup>

## Step-by-step

<Steps>
  <Step title="Install vm2">
    Add vm2 to your project using the installation command above. vm2 has no runtime dependencies beyond acorn (used internally for code transformation) and works with any Node.js version 6 or later.
  </Step>

  <Step title="Create a VM sandbox and run code">
    `VM` is the simpler of the two sandbox classes. It runs synchronous JavaScript with access only to built-in JavaScript objects and `Buffer`. There is no `require`, no `process`, and no access to the host environment unless you pass values in via `sandbox`.

    ```javascript
    import { VM } from 'vm2';

    const vm = new VM({
      timeout: 1000,
      allowAsync: false,
      sandbox: {},
    });

    vm.run('process.exit()'); // throws ReferenceError: process is not defined
    ```

    You can retrieve return values directly:

    ```javascript
    let number = vm.run('1337'); // returns 1337
    ```

    <Tip>
      Use `timeout` together with `allowAsync: false` for the strongest execution constraints. The `timeout` option only applies to synchronous code passed to `run` — it does not limit methods returned from the sandbox.
    </Tip>
  </Step>

  <Step title="Create a NodeVM sandbox with module access">
    `NodeVM` allows sandboxed code to `require` modules, similar to a normal Node.js script. You control exactly which built-in and external modules are allowed.

    ```javascript
    import { NodeVM } from 'vm2';

    const vm = new NodeVM({
      console: 'inherit',
      sandbox: {},
      require: {
        external: true,
        builtin: ['fs', 'path'],
        root: './',
        mock: {
          fs: {
            readFileSync: () => 'Nice try!',
          },
        },
      },
    });

    // Export a function from the sandbox and call it from the host
    let functionInSandbox = vm.run(
      'module.exports = function(who) { console.log("hello " + who); }'
    );
    functionInSandbox('world'); // logs: hello world
    ```

    You can also run scripts that use `require` directly:

    ```javascript
    import { NodeVM } from 'vm2';

    const vm = new NodeVM({
      require: {
        external: true,
        root: './',
      },
    });

    vm.run(
      `
      var request = require('request');
      request('http://www.google.com', function (error, response, body) {
          console.error(error);
          if (!error && response.statusCode == 200) {
              console.log(body);
          }
      });
      `,
      'vm.js',
    );
    ```

    <Note>
      The second argument to `vm.run()` in NodeVM is the filename. Pass the full path of the script when loading modules by relative path — it is used to resolve `require` calls and appears in stack traces.
    </Note>

    <Warning>
      `NodeVM` does not support the `timeout` option. It is not immune to infinite loops like `while (true) {}`. The more modules you allow, the larger your attack surface.
    </Warning>
  </Step>

  <Step title="Use VMScript for precompiled code">
    `VMScript` lets you compile a script once and run it multiple times, which improves performance when you execute the same code repeatedly. The compiled script is not bound to any particular sandbox — it is rebound at each `run` call.

    ```javascript
    import { VM, VMScript } from 'vm2';

    const vm = new VM();
    const script = new VMScript('Math.random()');

    console.log(vm.run(script)); // e.g. 0.4176842310
    console.log(vm.run(script)); // e.g. 0.9103211874
    ```

    `VMScript` also works with `NodeVM`:

    ```javascript
    import { NodeVM, VMScript } from 'vm2';

    const vm = new NodeVM();
    const script = new VMScript('module.exports = Math.random()');

    console.log(vm.run(script));
    console.log(vm.run(script));
    ```

    Code compiles automatically on first run. You can also trigger compilation explicitly:

    ```javascript
    try {
      const script = new VMScript('Math.random()').compile();
    } catch (err) {
      console.error('Failed to compile script.', err);
    }
    ```

    <Tip>
      Pre-compile scripts at application startup to avoid compilation cost on the first execution in production.
    </Tip>
  </Step>
</Steps>

## Error handling

Synchronous errors from compilation and execution can be caught with `try/catch`. Asynchronous errors require an `uncaughtException` handler on `process`:

```javascript
try {
  var script = new VMScript('Math.random()').compile();
} catch (err) {
  console.error('Failed to compile script.', err);
}

try {
  vm.run(script);
} catch (err) {
  console.error('Failed to execute script.', err);
}

process.on('uncaughtException', err => {
  console.error('Asynchronous error caught.', err);
});
```
